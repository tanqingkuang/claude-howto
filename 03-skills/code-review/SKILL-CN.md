# 代码审查技能

这个技能提供全面的代码审查能力，重点关注安全、性能、质量和可维护性，并且会在输出中尽量给出可执行的修复建议：

在审查时，优先找出会影响用户、数据安全、运行效率或后续维护成本的问题；如果存在多个问题，优先按“风险高、影响大、修复快”的顺序组织结论，方便用户快速采取行动。

1. **安全分析**
   - 身份验证 / 授权问题
   - 数据泄露风险
   - 注入漏洞
   - 密码学弱点
   - 敏感数据日志记录

2. **性能审查**
   - 算法效率（Big O 分析）
   - 内存优化
   - 数据库查询优化
   - 缓存机会
   - 并发问题和竞争条件

3. **代码质量**
   - SOLID 原则
   - 设计模式
   - 命名规范
   - 文档与注释
   - 测试覆盖率

4. **可维护性**
   - 代码可读性
   - 函数长度（建议少于 50 行）
   - 圈复杂度
   - 依赖管理
   - 类型安全

## 审查模板

对每一段被审查的代码，请提供：

### 摘要
- 整体质量评分（1-5）
- 关键发现数量
- 建议优先关注的区域
- 一句话结论，方便快速决策
- 如果需要，也可以给出“建议先改哪里”的最短路径

### 关键问题（如有）
- **问题**：清晰描述
- **位置**：文件和行号
- **影响**：为什么这很重要
- **严重性**：Critical / High / Medium
- **修复**：代码示例
- 如果可能，补充验证建议或测试思路
- 如果修复涉及多个文件，说明推荐的实施顺序

### 按类别列出发现

#### 安全性（如有问题）
列出安全漏洞及示例，并说明可能的攻击面或触发条件

#### 性能（如有问题）
列出性能问题，并说明复杂度、热路径或资源开销

#### 质量（如有问题）
列出代码质量问题，并给出重构建议、命名改进或结构调整建议

#### 可维护性（如有问题）
列出可维护性问题，并给出改进建议，帮助后续开发更容易理解和修改
